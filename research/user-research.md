## 1. 目标用户画像

## 3. 用户访谈大纲

### 3.1 访谈基本信息

- **访谈时长**：45 分钟
- **访谈形式**：一对一深度访谈（线上/线下）
- **访谈对象**：目标用户画像中的典型用户
- **访谈人数**：每类用户 3-5 人，共 15-25 人
- **记录方式**：录音 + 文字记录（需征得同意）

---

### 3.2 开场介绍（5 分钟）

**主持人话术：**

> "您好，感谢您参加今天的访谈。我是聚宝盆项目的研究员小 C。
> 
> 今天想和您聊聊关于手机游戏的一些想法和体验，整个过程大约 45 分钟。
> 
> 访谈内容仅用于产品研究，会严格保密，您可以放心畅所欲言。
> 
> 过程中如果您有任何问题或想暂停，随时告诉我。
> 
> 请问我可以开始录音吗？（等待同意）
> 
> 好的，那我们开始吧！"

**暖场问题：**
- 今天过来/上线方便吗？
- 平时工作/学习忙吗？

---

### 3.3 游戏习惯了解（10 分钟）

**目标**：了解用户真实的游戏行为和偏好

**问题列表：**

1. "能聊聊您平时玩游戏的情况吗？比如最近在玩什么游戏？"
   - 追问：为什么选择这款游戏？

2. "您一般什么时候玩游戏？每次大概玩多久？"
   - 追问：工作日和周末有区别吗？

3. "您印象最深的一款游戏是什么？为什么印象深刻？"
   - 追问：现在还在玩吗？如果不玩了，是什么原因放弃的？

4. "您有没有在一款游戏里付费过？方便说说是什么情况吗？"
   - 追问：付费后感觉值得吗？

5. "您会和朋友一起玩游戏吗？或者会推荐游戏给朋友？"
   - 追问：什么情况下会推荐？

**观察要点：**
- 用户提到游戏时的表情和语气
- 对付费话题的态度
- 社交游戏的参与度

---

### 3.4 产品概念测试（15 分钟）

**目标**：测试用户对"聚宝盆"概念的接受度和期待

**材料准备**：产品概念介绍（1-2 页 PPT 或原型演示）

**主持人话术：**

> "接下来我想给您看一个我们正在构思的游戏概念，想听听您的真实想法。
> 
> 这个游戏叫'聚宝盆'，核心玩法是……（简要介绍）"

**问题列表：**

1. "听完介绍，您对这个游戏的第一印象是什么？"
   - 追问：哪里最吸引您？哪里让您犹豫？

2. "如果这款游戏上线，您会有兴趣尝试吗？为什么？"
   - 追问：什么情况下会下载？什么情况下会放弃？

3. "您觉得'聚宝盆'这个主题怎么样？有什么联想吗？"
   - 追问：您希望看到什么样的视觉风格？

4. "假设游戏有以下玩法，您最期待哪个？"
   - 放置收获
   - 宝物收集
   - 随机惊喜
   - 社交互动

5. "您觉得这个游戏可能哪里会让您觉得无聊或想放弃？"
   - 追问：我们怎么改进会更好？

6. "如果让您给这个游戏提一个建议，您会说什么？"

**观察要点：**
- 用户对概念的真实反应（兴奋/平淡/疑惑）
- 对核心玩法的理解程度
- 自发提出的问题和建议

---

### 3.5 付费意愿探索（10 分钟）

**目标**：深入了解用户的付费心理和接受度

**问题列表：**

1. "刚才介绍的游戏里，如果有付费内容，您觉得什么样的付费您能接受？"
   - 追问：什么样的付费您绝对不能接受？

2. "您觉得一款游戏定价多少算合理？比如月卡、礼包之类的。"
   - 追问：您心理价位是多少？

3. "假设游戏有以下付费选项，您最可能考虑哪个？"
   - 展示付费方案 A/B/C
   - 追问：为什么选这个？

4. "您有没有过'付费后后悔'的经历？是什么情况？"
   - 追问：什么情况下会觉得"这钱花得值"？

5. "如果游戏完全免费但有广告，和付费去广告，您更倾向哪种？"
   - 追问：能接受多少广告？

6. "您觉得游戏里什么情况下会让您想付费？"
   - 限时优惠？
   - 稀缺道具？
   - 社交压力？
   - 其他？

**观察要点：**
- 用户对付费话题的敏感度
- 价格敏感区间
- 付费触发因素

---

### 3.6 总结（5 分钟）

**目标**：收尾并获取额外反馈

**问题列表：**

1. "今天我们聊了很多，您还有什么想补充的吗？"
   - 关于游戏、付费、或任何想法

2. "如果游戏上线后邀请您参与测试，您愿意吗？"
   - 追问：希望我们如何联系您？

3. "您身边有朋友可能对这类游戏感兴趣吗？方便推荐他们参与调研吗？"

**结束语：**

> "非常感谢您今天的分享，对我们帮助很大！
> 
> 后续如果有任何进展，我们会通过（微信/邮件）与您联系。
> 
> 这是给您的小礼物/报酬，请收下。
> 
> 再次感谢，祝您生活愉快！"

**访谈后记录：**
- 用户整体态度（积极/中性/消极）
- 关键洞察（3-5 条）
- 需要跟进的问题

## 4. A/B 测试方案

### 4.1 测试变量设计

#### 测试 1：新手引导流程

| 版本 | 设计 | 假设 |
|------|------|------|
| A 版 | 传统分步引导（5 分钟） | 清晰但可能冗长 |
| B 版 | 沉浸式体验引导（3 分钟） | 更快进入核心玩法 |

### 4.2 样本量计算

**计算公式**：
```
n = (Zα/2 + Zβ)² × (p1(1-p1) + p2(1-p2)) / (p1 - p2)²
```

**参数设定**：
- 显著性水平 α = 0.05（Zα/2 = 1.96）
- 统计功效 1-β = 0.8（Zβ = 0.84）
- 基准转化率 p1 = 5%
- 期望提升 p2 = 6%（相对提升 20%）

**计算结果**：
- 每组最小样本量：**约 2,400 人**
- 考虑 10% 流失，建议每组：**2,700 人**

**测试周期估算**：
- 日新增用户：500 人
- 分配到 2 组：每组 250 人/天
- 达到样本量所需时间：**约 11 天**

### 4.3 指标定义

#### 核心指标

| 指标 | 定义 | 计算方式 |
|------|------|----------|
| DAU | 日活跃用户数 | 当日登录去重用户数 |
| WAU | 周活跃用户数 | 当周登录去重用户数 |
| MAU | 月活跃用户数 | 当月登录去重用户数 |
| 次日留存 | 新用户第 2 日留存率 | D1 登录用户数 / D0 新增用户数 |
| 7 日留存 | 新用户第 7 日留存率 | D7 登录用户数 / D0 新增用户数 |
| 30 日留存 | 新用户第 30 日留存率 | D30 登录用户数 / D0 新增用户数 |

#### 付费指标

| 指标 | 定义 | 计算方式 |
|------|------|----------|
| 付费率 | 付费用户占比 | 付费用户数 / 活跃用户数 |
| ARPU | 每用户平均收入 | 总收入 / 活跃用户数 |
| ARPPU | 每付费用户平均收入 | 总收入 / 付费用户数 |
| LTV | 用户生命周期价值 | 用户整个生命周期的贡献收入 |

#### 传播指标

| 指标 | 定义 | 计算方式 |
|------|------|----------|
| 分享率 | 分享用户占比 | 分享用户数 / 活跃用户数 |
| K 因子 | 病毒系数 | 每个用户带来的新用户数 |
| 邀请转化率 | 邀请成功比例 | 通过邀请新增用户 / 发出邀请数 |

```
┌─────────────────────────────────────────────────────────┐
│                    A/B 测试完整流程                      │
└─────────────────────────────────────────────────────────┘

第 1 周：准备阶段

└─ 准备数据埋点和监控

第 2-3 周：执行阶段
├─ 小流量灰度（5% 用户）
├─ 观察异常，确保稳定性
├─ 扩大流量至 50%/50%
├─ 持续监控核心指标
└─ 收集定性反馈（用户访谈）

第 4 周：分析阶段
├─ 数据清洗和整理

├─ 细分群体分析
├─ 撰写测试报告
└─ 决策：全量/迭代/放弃

后续：
├─ 全量获胜版本
├─ 归档测试文档
└─ 规划下一轮测试
```

**决策标准**：
- 统计显著（p < 0.05）且指标提升 >5% → 全量
- 统计显著但提升 <5% → 讨论是否值得
- 统计不显著 → 延长测试或放弃

### 5.2 数据收集方法

3. **异常行为监控**：崩溃、卡顿、错误

### 5.3 分析模型

#### 1. 漏斗分析（Funnel Analysis）

**用途**：分析用户转化路径，定位流失点

**示例**：新手转化漏斗
```
启动 App → 完成注册 → 完成新手引导 → 首次核心玩法 → 次日留存
  100%      85%         60%              45%            40%
```

**行动**：针对流失率高的环节优化

---

#### 2. 留存分析（Retention Analysis）

**用途**：分析用户留存规律，评估产品健康度

**分析方法**：
- 同期群分析（Cohort Analysis）
- 留存曲线对比
- 留存用户行为特征分析

**示例**：
```
         D0   D1   D2   D3   D4   D5   D6   D7
第 1 周   100% 45%  35%  30%  28%  26%  25%  24%
第 2 周   100% 42%  33%  28%  26%  25%  24%  23%
第 3 周   100% 40%  32%  27%  25%  24%  23%  22%
```

---

#### 3. 用户分群（Segmentation）

**用途**：精细化运营，针对性优化

**分群维度**：
- 人口属性：年龄、性别、地区
- 行为特征：活跃时段、玩法偏好
- 价值分层：免费用户、小额付费、大额付费
- 生命周期：新用户、活跃用户、流失用户

**应用**：
- 不同群体推送不同内容
- 高价值用户专属服务
- 流失用户召回策略

---

#### 4. LTV 预测模型

**用途**：预测用户生命周期价值，指导获客预算

**简化模型**：
```
LTV = ARPU × 平均生命周期（月）
平均生命周期 = 1 / (1 - 月留存率)
```

**进阶模型**：
- 基于机器学习的预测（随机森林、XGBoost）
- 考虑用户行为序列（RNN、LSTM）

---

#### 5. AARRR 模型

```
┌──────────────────────────────────────────────────────┐
│                    AARRR 增长模型                     │
├──────────────────────────────────────────────────────┤
│  Acquisition（获取）→ Activation（激活）→            │
│  Retention（留存）→ Revenue（收入）→                 │
│  Referral（推荐）                                    │
└──────────────────────────────────────────────────────┘
```

**各阶段核心指标**：

| 阶段 | 核心问题 | 关键指标 |
|------|----------|----------|
| 获取 | 用户从哪里来？ | 渠道 ROI、CAC |
| 激活 | 用户体验到价值了吗？ | 新手完成率、核心行为率 |
| 留存 | 用户会回来吗？ | 次日/7 日/30 日留存 |
| 收入 | 用户愿意付费吗？ | 付费率、ARPU、LTV |
| 推荐 | 用户会推荐吗？ | K 因子、分享率 |

---

#### 6. RFM 用户价值模型

| 维度 | 含义 | 应用 |
|------|------|------|
| R (Recency) | 最近一次消费时间 | 识别活跃用户 |
| F (Frequency) | 消费频率 | 识别忠诚用户 |
| M (Monetary) | 消费金额 | 识别高价值用户 |

**用户分层**：
- 重要价值用户（高 R、高 F、高 M）
- 重要发展用户（高 R、低 F、高 M）
- 重要保持用户（低 R、高 F、高 M）
- 一般用户（其他）
