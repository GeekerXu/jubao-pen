# 贡献指南

感谢你关注聚宝盆项目！我们欢迎各种形式的贡献。

## 🤝 如何参与项目

### 1.  Fork 仓库

点击 GitHub 页面右上角的 "Fork" 按钮，创建你自己的副本。

### 2. 克隆仓库

```bash
git clone https://github.com/your-username/jubao-pen.git
cd jubao-pen
```

### 3. 创建分支

```bash
git checkout -b feature/your-feature-name
```

分支命名规范：
- `feature/xxx` - 新功能
- `fix/xxx` - 问题修复
- `docs/xxx` - 文档更新
- `research/xxx` - 调研内容

### 4. 进行贡献

你可以：
- 📝 撰写或完善项目计划
- 🔍 进行深度调研并提交报告
- 🎨 设计 HTML 页面模板
- 🐛 修复问题或提出改进建议
- 📚 完善文档

### 5. 提交更改

```bash
git add .
git commit -m "feat: 添加 xxx 项目计划"
git push origin feature/your-feature-name
```

### 6. 创建 Pull Request

在 GitHub 上提交 Pull Request，描述你的更改内容。

## 📋 代码规范

### 文档规范

**Markdown 格式**：
```markdown
# 一级标题

## 二级标题

### 三级标题

- 列表项 1
- 列表项 2

**粗体文本**
*斜体文本*

[链接文本](https://example.com)

![图片描述](image.png)
```

**代码块**：
```markdown
```javascript
// 标注语言类型
const hello = "world";
```
```

### 文件命名

- ✅ 正确：`project-plan-001.md`
- ✅ 正确：`research-report.md`
- ❌ 错误：`Project Plan 001.md`
- ❌ 错误：`project_plan_001.md`

### 内容规范

1. **项目计划**应包含：
   - 项目背景
   - 研究目标
   - 关键问题
   - 预期成果
   - 时间规划

2. **调研报告**应包含：
   - 研究背景
   - 方法论
   - 核心发现
   - 结论与建议
   - 参考资料

3. **HTML 页面**应遵循：
   - 语义化标签
   - 响应式设计
   - 无障碍访问
   - 性能优化

## 🔄 PR 流程

### 1. 提交 PR

- 填写清晰的标题
- 描述更改内容
- 关联相关 Issue（如有）
- 添加适当的标签

### 2. Code Review

- 维护者会审查代码
- 可能需要修改完善
- 保持友好沟通

### 3. 合并

- 审查通过后合并
- 自动部署到 GitHub Pages
- 关闭相关 Issue

### PR 模板

```markdown
## 更改类型
- [ ] 新功能
- [ ] 问题修复
- [ ] 文档更新
- [ ] 其他

## 描述
简要描述此 PR 的内容。

## 相关 Issue
Fixes #123

## 检查清单
- [ ] 代码/文档符合规范
- [ ] 添加了必要的测试（如适用）
- [ ] 更新了相关文档
- [ ] 本地测试通过
```

## 💡 贡献建议

### 新手入门

1. 从 `good first issue` 标签的问题开始
2. 阅读现有文档了解项目结构
3. 在 Issue 中留言确认你要处理的问题
4. 不要害怕提问

### 进阶贡献

1. 主导一个完整的项目调研
2. 改进 HTML 模板设计
3. 优化项目结构
4. 帮助审查其他贡献者的 PR
