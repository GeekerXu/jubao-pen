# 项目 012: 好友默契度测试 - 详细方案

## 📖 项目概述

**项目名称**: 好友默契度测试 (Friendship Compatibility Test)

**一句话介绍**: 你和好友有多默契？答题测试，生成默契报告

**推荐优先级**: Top 3

**核心价值**: 利用社交互动和炫耀心理，打造病毒式传播的测试类应用。

---

## 🎮 详细玩法说明

### 核心流程

```
创建测试 → 分享给好友 → 双方答题 → AI 计算默契度 → 生成报告 → 分享炫耀
```

### 详细规则

#### 1. 测试创建
- 用户选择测试类型：
  - 基础版：10 道题（免费）
  - 进阶版：30 道题（付费）
  - 定制版：自定义题目（付费）
  
- 题目类型：
  - 关于对方（"TA 最喜欢的颜色是？"）
  - 共同回忆（"第一次见面在哪里？"）
  - 价值观（"更看重事业还是家庭？"）
  - 情景假设（"吵架后谁先道歉？"）

#### 2. 答题机制
- 创建者先答题（设定"标准答案"）
- 分享链接给好友
- 好友独立答题
- 双方都完成后解锁结果

#### 3. 默契度计算
- 每题权重不同（核心问题权重高）
- AI 分析答案相似度
- 生成百分比分数（0-100%）
- 分级评价：
  - 90-100%: 灵魂伴侣
  - 80-89%: 最佳好友
  - 70-79%: 默契十足
  - 60-69%: 普通朋友
  - <60%: 还需要了解

#### 4. 报告生成
- 总体默契度分数
- 各维度分析（了解度/价值观/习惯等）
- 亮点题目展示
- 改进建议
- 精美可视化图表

### 社交传播
- 分享报告到朋友圈
- 邀请更多人测试（"测测我们有多默契"）
- 默契度排行榜（好友间）
- 生成默契证书（可下载）

## 🏗️ 技术架构

### 技术栈
- 前端：微信小程序
- 后端：Node.js + Express
- 数据库：MongoDB（灵活存储题目和报告）
- 缓存：Redis（排行榜）

### 核心 API
```javascript
POST /api/v1/test/create
Body: { questions, friendWechat }

POST /api/v1/test/submit
Body: { testId, answers }

GET /api/v1/test/result/:testId
Response: { score, analysis, report }
```
