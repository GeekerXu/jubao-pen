# 项目 011: 省份战力排行榜 - 详细方案

## 📖 项目概述

**项目名称**: 省份战力排行榜 (Province Power Ranking)

**一句话介绍**: 类似羊了个羊，按省份排名，为家乡而战

**推荐优先级**: Top 2

**核心价值**: 利用地域荣誉感和从众心理，打造病毒式传播的社交游戏，参考羊了个羊成功模式。

---

## 🎮 详细玩法说明

### 核心玩法

**游戏类型**: 简单消除/闯关玩法

**核心循环**:
```
选择省份 → 开始闯关 → 通关计入省份分数 → 查看排名 → 分享拉人
```

### 详细规则

#### 1. 省份选择
- 用户首次进入选择家乡省份（34 个省级行政区）
- 选择后不可更改（增加仪式感）
- 显示当前省份排名和战力

#### 2. 闯关玩法
- **关卡设计**: 三消变体，类似羊了个羊
  - 屏幕上方堆叠各种图案的方块
  - 下方 7 个卡槽位
  - 点击方块落入卡槽，3 个相同消除
  - 卡槽满则游戏失败
  
- **关卡难度**:
  - 每日 3 关，难度递增
  - 第 1 关：教学关，90% 通过率
  - 第 2 关：中等难度，50% 通过率
  - 第 3 关：地狱难度，<10% 通过率（制造话题）

- **道具系统**:
  - 移出卡：移出 3 个方块到临时位置
  - 撤回卡：撤回最后一步
  - 洗牌卡：重新排列剩余方块
  - 获取方式：看广告/好友助力/每日免费

#### 3. 排名系统
- **省份排行榜**:
  - 实时显示 34 省份排名
  - 排名依据：通关人数 × 关卡权重
  - 每小时更新一次
  
- **个人贡献**:
  - 显示个人为省份贡献的分数
  - 省内排名（市/县级别）
  - 贡献勋章系统

#### 4. 社交机制
- **助力系统**:
  - 分享链接给好友，好友点击获得道具
  - 每日最多 3 次助力
  - 助力者也为对方省份 +1 分（双赢）
  
- **炫耀分享**:
  - 通关后生成分享卡片
  - "我为 XX 省贡献了 XX 分，当前排名全国第 X"
  - 卡片带省份特色元素

- **省份群聊**:
  - 同省用户可加入"省群"
  - 群内讨论攻略、互相助力
  - 群排行榜（省内各市 PK）

### 特殊活动

#### 省份 PK 赛
- 每周选定两个相邻省份 PK
- PK 期间通关分数×2
- 输的省份"挂彩旗"（趣味惩罚）

#### 节日活动
- 春节：各省特色年俗方块
- 国庆：红色主题关卡
- 中秋：月饼消除图案

### 防作弊机制
- 设备指纹识别
- 通关时间检测（过快判定作弊）
- IP 异常监控
- 作弊者分数清零

### 核心数据结构

#### Redis 排行榜设计

```redis
# 省份总分排行榜（有序集合）
ZSET province_rank
Key: province_rank:date:2026-03-03
Member: 省份 ID, Score: 总分

# 省内用户排行
ZSET province_user_rank:{province_id}
Member: user_id, Score: 个人贡献分

# 关卡通过率统计
HASH level_stats:level:3
Key: level_stats:level:3
Field: total_attempts, success_count, avg_time

# 用户每日通关状态
HASH user_daily:{user_id}:2026-03-03
Field: level_1, level_2, level_3 (0/1)
```

#### MySQL 表设计

```sql
-- 用户表
CREATE TABLE users (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  openid VARCHAR(64) UNIQUE,
  province_id TINYINT,
  nickname VARCHAR(50),
  avatar_url VARCHAR(255),
  total_score INT DEFAULT 0,
  created_at TIMESTAMP,
  INDEX idx_province (province_id)
);

-- 关卡表
CREATE TABLE levels (
  id INT PRIMARY KEY,
  date DATE,
  level_num TINYINT,
  layout JSON,  -- 方块布局配置
  pass_rate DECIMAL(5,2),
  created_at TIMESTAMP
);

-- 通关记录表
CREATE TABLE pass_records (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT,
  level_id INT,
  province_id TINYINT,
  pass_time INT,  -- 用时（秒）
  used_props JSON,
  created_at TIMESTAMP,
  INDEX idx_user_level (user_id, level_id),
  INDEX idx_province_date (province_id, created_at)
);

-- 省份日统计表
CREATE TABLE province_daily_stats (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  province_id TINYINT,
  stat_date DATE,
  total_pass INT,
  total_score INT,
  rank INT,
  UNIQUE KEY uk_province_date (province_id, stat_date)
);
```

### 并发处理策略

#### 排行榜更新
```go
// 使用 Redis 原子操作，避免锁竞争
func addProvinceScore(provinceId int, score int) {
    redis.ZIncrBy("province_rank", score, provinceId)
    
    // 异步更新 MySQL 统计
    kafka.Publish("province_score_event", {
        province_id: provinceId,
        score: score,
        timestamp: time.Now()
    })
}

// 定时任务：每小时固化排行榜
func snapshotRanking() {
    ranks := redis.ZRevRangeWithScores("province_rank", 0, -1)
    saveToMySQL(ranks)
}
```

#### 防刷机制
```go
// 设备指纹 + 频率限制
func checkCheat(userId string, deviceId string) bool {
    // 1. 检查设备当日通关次数
    count := redis.Get("device_pass_count:" + deviceId)
    if count > 50 {
        return true  // 异常
    }
    
    // 2. 检查通关时间
    if passTime < 30 {  // 少于 30 秒判定作弊
        return true
    }
    
    // 3. IP 聚集检测
    ipCount := redis.SCard("ip_users:" + ip)
    if ipCount > 10 {
        return true  // 同一 IP 过多用户
    }
    
    return false
}
```
