# 项目 003: AI 猜词大师 - 详细方案

## 📖 项目概述

**项目名称**: AI 猜词大师 (AI Word Guess Master)

**一句话介绍**: AI 出题你来猜，或者你描述 AI 猜，双向互动的猜词游戏

**推荐优先级**: Top 3

**核心价值**: 结合 AI 生成能力和经典猜词玩法，提供无限词库和智能提示，适合朋友聚会、亲子互动、语言学习等多场景。

---

## 🎮 详细玩法说明

### 核心玩法模式

#### 模式 1: AI 出题·你来猜

**核心流程**:
```
选择难度/类别 → AI 生成词语 → 显示字数/类别提示 → 用户输入猜测 → AI 给予提示 → 猜对/放弃
```

**详细规则**:
- **难度分级**:
  - 简单：常见词汇，3-4 字，提示丰富
  - 中等：成语/俗语，4-5 字，适度提示
  - 困难：生僻词/专业术语，提示较少
  - 地狱：随机组合，极少提示

- **提示系统**:
  - 字数提示：显示"_ _ _ _"格式
  - 类别提示：如"动物"、"成语"、"明星"
  - 智能提示：AI 生成相关线索（消耗提示币）
  - 首字提示：显示第一个字（消耗提示币）
  - 排除提示：AI 告知"不包含 X 字"

- **计分规则**:
  - 首次猜对：100 分
  - 每次提示：-20 分
  - 超时未猜出：0 分
  - 连续猜对：连击加成

#### 模式 2: 你描述·AI 来猜

**核心流程**:
```
用户心中想词 → 开始游戏 → 用户描述（不能含目标字）→ AI 提问/猜测 → 用户反馈 → AI 继续猜
```

**详细规则**:
- **描述限制**:
  - 不能说目标词中的任何字
  - 不能谐音暗示
  - 每次描述限 50 字以内
  - AI 会检测违规描述

- **AI 猜测策略**:
  - 基于描述提取关键词
  - 主动提问缩小范围
  - 每轮给出 1-3 个猜测
  - 最多 10 轮猜中为目标

- **评分标准**:
  - AI 5 轮内猜中：⭐⭐⭐⭐⭐
  - AI 10 轮内猜中：⭐⭐⭐⭐
  - AI 未猜中：用户胜

#### 模式 3: 双人 PK 模式

**核心流程**:
```
玩家 A 描述 → 玩家 B 猜 → 计时计分 → 交换角色 → 比较分数
```

**详细规则**:
- 每轮 3 分钟限时
- 描述者不能说出目标字
- 猜词者无限次猜测
- 最终比较猜对数量

### 进阶玩法

#### 主题词库
- **影视娱乐**: 电影名、明星、角色
- **历史文化**: 历史人物、典故、文物
- **地理知识**: 国家、城市、景点
- **科学知识**: 科学术语、发明家、公式
- **网络流行**: 梗、流行语、网红

#### 限时挑战
- **60 秒冲刺**: 60 秒内猜对尽可能多的词
- **每日挑战**: 每日固定 5 词，全球排名
- **主题周赛**: 每周一个主题，累计积分

#### 成就系统
- "猜词新手": 首次猜对
- "词库达人": 累计猜对 100 词
- "提示大师": 不用提示连续猜对 10 次
- "火眼金睛": 10 秒内快速猜对

### 用户旅程示例

```
新用户:
下载 → 选择"AI 出题"模式 → 简单难度 → 猜对 3 词 → 
解锁成就 → 分享成绩 → 尝试"你描述 AI 猜" → 觉得有趣 → 
邀请朋友 PK → 形成习惯

老用户:
每日登录 → 完成每日挑战 → 查看排名 → 
参与主题周赛 → 购买提示币 → 解锁高级词库 → 
创建自定义词库 → 分享给朋友
```

### 核心 API 设计

```typescript
// 1. 开始游戏
POST /api/v1/game/start
Body: { mode: "ai_guess" | "user_describe" | "pk", difficulty, category }
Response: {
  gameId: string,
  word: string (仅服务端存储),
  hints: { length, category, initialHint },
  timeLimit: number
}

// 2. 提交猜测
POST /api/v1/game/guess
Body: { gameId, guessWord }
Response: {
  isCorrect: boolean,
  newHint?: string,
  score: number,
  remainingTime: number
}

// 3. AI 猜测（你描述模式）
POST /api/v1/game/ai-guess
Body: { gameId, userDescription, isCorrect: boolean }
Response: {
  aiGuess: string,
  aiQuestion?: string,
  confidence: number,
  round: number
}

// 4. 获取提示
POST /api/v1/game/hint
Body: { gameId, hintType: "smart" | "first_char" | "exclude" }
Response: {
  hint: string,
  cost: number
}
```

### AI Prompt 设计

#### AI 出题 Prompt
```
你是一位猜词游戏出题者。请根据以下要求生成词语：

难度：{difficulty}
类别：{category}
已使用词语：{used_words}

要求：
1. 生成一个符合难度和类别的词语
2. 不要与已使用词语重复
3. 提供 3 个渐进式提示（从模糊到具体）
4. 输出格式：
{
  "word": "词语",
  "length": 字数，
  "hints": ["提示 1", "提示 2", "提示 3"],
  "category": "类别"
}
```

#### AI 猜词 Prompt
```
你正在玩猜词游戏。用户心中想了一个词，你需要通过提问和猜测来找出它。

对话历史：{conversation_history}
用户描述：{user_description}
已猜测词语：{guessed_words}

策略：
1. 分析用户描述中的关键词
2. 缩小范围（类别、特征、用途等）
3. 提出有针对性的问题或直接猜测
4. 避免重复已猜测的词语

输出格式：
{
  "guess": "你的猜测",
  "question": "你想问的问题（可选）",
  "confidence": 0.8,
  "reasoning": "猜测理由"
}
```

### 数据库设计

```sql
-- 用户表
CREATE TABLE users (
  id VARCHAR(32) PRIMARY KEY,
  nickname VARCHAR(50),
  avatar_url VARCHAR(255),
  total_score INT DEFAULT 0,
  created_at TIMESTAMP
);

-- 游戏记录表
CREATE TABLE game_records (
  id VARCHAR(32) PRIMARY KEY,
  user_id VARCHAR(32),
  mode VARCHAR(20),
  difficulty VARCHAR(20),
  category VARCHAR(50),
  score INT,
  words_guessed INT,
  hints_used INT,
  duration INT,
  created_at TIMESTAMP,
  INDEX idx_user (user_id),
  INDEX idx_date (created_at)
);

-- 词库表
CREATE TABLE word_bank (
  id INT PRIMARY KEY AUTO_INCREMENT,
  word VARCHAR(50),
  category VARCHAR(50),
  difficulty VARCHAR(20),
  hints JSON,
  usage_count INT DEFAULT 0
);

-- 成就表
CREATE TABLE achievements (
  id VARCHAR(32) PRIMARY KEY,
  user_id VARCHAR(32),
  achievement_type VARCHAR(50),
  unlocked_at TIMESTAMP,
  UNIQUE KEY uk_user_achievement (user_id, achievement_type)
);
```
