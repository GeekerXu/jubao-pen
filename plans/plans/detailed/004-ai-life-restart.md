# 项目 004: AI 人生重开模拟器 - 详细方案

## 📖 项目概述

**项目名称**: AI 人生重开模拟器 (AI Life Restart Simulator)

**一句话介绍**: 如果可以重来，你的人生会怎样？AI 生成无限可能的人生剧本

**推荐优先级**: Top 3

**核心价值**: 结合热门"人生重开"玩法与 AI 生成能力，每次重开都是独特人生体验，满足用户好奇心、探索欲和社交分享需求。

---

## 🎮 详细玩法说明

### 核心流程

```
天赋抽取 → 属性分配 → 人生历程（0-100 岁）→ 人生总结 → 分享/重开
```

### 详细玩法

#### 1. 天赋抽取系统

**天赋类型**:
- **SSR 级天赋** (1% 概率): 
  - "天选之子"：所有事件向好发展
  - "财富自由"：初始资金×10
  - "天才智商"：学习/事业成功率 +50%
  
- **SR 级天赋** (5% 概率):
  - "运动健将"：健康 +20%
  - "艺术细胞"：艺术相关事件触发率 +30%
  - "社交达人"：人际关系事件向好
  
- **R 级天赋** (30% 概率):
  - "早睡早起"：健康小幅提升
  - "勤俭节约"：资金消耗 -10%
  
- **N 级天赋** (64% 概率):
  - 普通天赋，轻微影响

**抽取规则**:
- 初始可抽取 10 个天赋
- 选择 3 个保留
- 可用"重随"道具刷新（看广告/付费）

#### 2. 属性分配

**四项基础属性**（初始共 20 点，每项上限 10）:
- **颜值**: 影响社交、恋爱事件
- **智力**: 影响学习、事业事件
- **体质**: 影响健康、寿命
- **家境**: 影响初始资源、机遇

**天赋影响**:
- 某些天赋可突破属性上限
- 某些天赋可初始赠送属性点

#### 3. 人生历程生成

**年龄阶段**:
- **0-6 岁 幼儿期**: 家庭事件、健康问题
- **7-18 岁 学生时代**: 学习、友谊、初恋
- **19-30 岁 青年期**: 大学、工作、恋爱、结婚
- **31-60 岁 中年期**: 事业、家庭、子女
- **61-100 岁 老年期**: 退休、健康、回忆

**事件类型**:
- **固定事件**: 每个人都会经历（如上学、高考）
- **条件事件**: 满足属性/天赋触发（如创业、出国）
- **随机事件**: 完全随机（如中彩票、遇贵人）
- **选择事件**: 用户做决定影响走向

**AI 生成机制**:
- 基于当前属性、天赋、历史事件生成下一年事件
- 保持逻辑连贯性（如学过音乐可能成为音乐家）
- 融入时代背景（如 90 后、00 后不同经历）

#### 4. 人生总结

**统计维度**:
- 寿命：活到多少岁
- 财富：巅峰资产
- 成就：职业高度
- 幸福：综合评分
- 特殊成就：如"环游世界"、"诺贝尔奖"

**人生评级**:
- S 级：传奇人生（前 1%）
- A 级：精彩人生（前 10%）
- B 级：平凡人生（前 50%）
- C 级：遗憾人生（后 40%）

### 进阶玩法

#### 特殊模式

**1. 平行宇宙模式**
- 在关键节点保存存档
- 可回到存档点重新选择
- 对比不同选择的人生轨迹

**2. 家族传承模式**
- 人生结束后可指定子女继承
- 部分天赋/属性可遗传
- 多代人家族发展史

**3. 历史穿越模式**
- 选择不同年代出生（60 后 -10 后）
- 体验不同时代的人生
- 收集各时代特殊成就

**4. 挑战模式**
- 极限挑战：0 属性开局
- 特定目标：如"30 岁赚 1 亿"
- 成就解锁：特殊条件通关

#### 社交玩法

**1. 人生分享**
- 生成人生时间轴长图
- 关键事件卡片
- 二维码分享好友体验

**2. 人生 PK**
- 好友间对比人生数据
- "谁的人生更精彩"投票
- 排行榜：首富、最长寿、最幸福

**3. 人生评论**
- 查看他人人生经历
- 留言互动
- "如果是我会..."讨论

### 用户旅程示例

```
第一次体验:
下载 → 抽取天赋（抽到 SSR 兴奋）→ 分配属性 → 
开始人生 → 20 岁遇到关键选择 → 纠结 → 
35 岁事业有成 → 60 岁退休 → 85 岁去世 → 
看人生总结（A 级）→ 分享朋友圈 → 想再玩一次

深度用户:
每日登录 → 领取免费重随 → 挑战特殊模式 → 
尝试不同年代 → 收集成就 → 培养家族 → 
查看排行榜 → 评论他人人生 → 社区互动
```

### 核心 API 设计

```typescript
// 1. 抽取天赋
POST /api/v1/game/talent-draw
Response: {
  talents: Array<{
    id: string,
    name: string,
    rarity: 'SSR' | 'SR' | 'R' | 'N',
    description: string,
    effects: object
  }>,
  rerollCount: number
}

// 2. 开始人生
POST /api/v1/game/start
Body: { selectedTalents, attributes }
Response: {
  gameId: string,
  initialStatus: object
}

// 3. 获取下一年事件
GET /api/v1/game/next-year/:gameId
Response: {
  age: number,
  event: {
    type: 'fixed' | 'conditional' | 'random' | 'choice',
    title: string,
    description: string,
    choices?: Array<{text, impact}>
  },
  status: { health, mood, assets, ... }
}

// 4. 提交选择
POST /api/v1/game/choice
Body: { gameId, choiceId }
Response: {
  result: string,
  statusChange: object,
  isAlive: boolean
}

// 5. 人生总结
GET /api/v1/game/summary/:gameId
Response: {
  lifespan: number,
  rating: 'S' | 'A' | 'B' | 'C',
  achievements: Array,
  statistics: object
}
```

### 数据库设计

```sql
-- 用户表
CREATE TABLE users (
  id VARCHAR(32) PRIMARY KEY,
  openid VARCHAR(64),
  nickname VARCHAR(50),
  created_at TIMESTAMP
);

-- 游戏存档表
CREATE TABLE game_saves (
  id VARCHAR(32) PRIMARY KEY,
  user_id VARCHAR(32),
  status VARCHAR(20), -- ongoing, completed
  talents JSON,
  attributes JSON,
  current_age INT,
  history JSON,
  created_at TIMESTAMP,
  updated_at TIMESTAMP,
  INDEX idx_user (user_id)
);

-- 人生记录表
CREATE TABLE life_records (
  id VARCHAR(32) PRIMARY KEY,
  user_id VARCHAR(32),
  lifespan INT,
  rating VARCHAR(5),
  achievements JSON,
  statistics JSON,
  share_code VARCHAR(20) UNIQUE,
  created_at TIMESTAMP,
  INDEX idx_user (user_id),
  INDEX idx_rating (rating)
);

-- 成就表
CREATE TABLE achievements (
  id VARCHAR(32) PRIMARY KEY,
  user_id VARCHAR(32),
  achievement_id VARCHAR(50),
  unlocked_at TIMESTAMP,
  UNIQUE KEY uk_user_achievement (user_id, achievement_id)
);
```
